# 02 CORE — Прятать и показывать иконки (спейсер)

Дата создания: 2026-06-05
Завершён: 2026-06-05 (ШАГ 1, проверено вживую на Tahoe)

## Цель

Реализовать ядро скрытия: свой невидимый разделитель-`NSStatusItem` раздувается и выталкивает соседние иконки за край экрана; клик на `<` возвращает их. Это путь HiddenBar/nanobar — ноль прав, надёжно на Tahoe. Чужие иконки не трогаем (см. ADR 003/004).

## Объём работ

- [x] Второй `NSStatusItem` — спейсер-cutter. На ШАГЕ 1 он ВИДИМЫЙ (полоска `▏`, 12pt) для отладки порядка; невидимым станет позже.
- [x] Скрытие: раздуть спейсер до ~ширины экрана — он выталкивает иконки левее себя за край.
- [x] Показ: вернуть спейсеру малую `length` — иконки уезжают обратно.
- [x] Состояние «скрыто / показано» и переключение по клику на якорь `<` (символ меняется `<` ↔ `v`).
- [x] GUARD порядка: раздуваем спейсер ТОЛЬКО если он реально левее якоря (сверка X-координат). Иначе скрытие блокируется (`⚠<`).
- [x] Стартовое авто-скрытие через `NSTimer` 1с после запуска (если guard пускает).
- [x] Позиции айтемов запоминаются через `setAutosaveName` (переживают перезапуск).

## Критерий готовности

Клик по якорю реально прячет лишние иконки за край экрана; повторный клик возвращает их. Скрытие работает только своим спейсером — чужие иконки не перерисовываются и не фотографируются.

## Итог (ШАГ 1)

Заработало, но НЕ так, как планировалось вначале. Расхождения с исходным планом (важны для следующих шагов):

- **Не один айтем, а два с явным GUARD.** Исходный план опирался на «спейсер создан вторым → окажется левее». На Tahoe порядок айтемов НЕ детерминирован порядком создания, и нет API двигать айтемы программно. Надёжность даёт guard: перед раздуванием сверяем реальные X обоих айтемов (`window().frame().origin.x`), раздуваем только если спейсер строго левее якоря. Иначе якорь показывает `⚠<` и тултип просит поправить Cmd+drag. Это убрало баг «`<` улетел за край». Подробности и поправка — research journal `003-status-item-order-and-hide.md` и ADR 005 (раздел «Поправка по итогам ШАГА 1»).
- **Координаты доступны отложенно** (~0.5–1с после старта, после стабилизации RunLoop) — отсюда стартовый таймер 1с, а не мгновенное скрытие.
- **Ширина скрытия ограничена** (clamp 500..4000, ширина экрана + запас 200). Гигантские значения (10000 из плана) дают патологию layout на Tahoe — приём подсмотрен у HiddenBar.
- **`defaults delete` на старте УБРАН** — он стирал запомненные через `setAutosaveName` позиции.
- **Спейсер пока видимый** (полоска `▏`) — временно, для отладки порядка. В коде помечено DEBUG.

Своя реализация, не копия `nanobar`. Код: `src/controller.rs` (toggle + guard + таймер), `src/status_bar.rs` (создание пары айтемов).
