# Правила ведения спринтов

## Нейминг

```
NN_CATEGORY_short-name.md
```

- `NN` — двузначный номер: `01`, `06` (не `1`).
- `CATEGORY` — заглавными: `CORE` / `UI` / `API` / `DATA` / `DEVOPS` / `TEST` / `DOCS` / `FIX`.
- `short-name` — строчными через дефис, по-английски: `status-item-ffi`.
- Дата — только ВНУТРИ файла, не в имени.

## Жизненный цикл

```
Создан        → sprints/planned/
Взят в работу → planned/ → active/
Завершён      → active/ → archive/
Отменён       → planned/ или active/ → deprecated/
```

`archive/` (завершено) ≠ `deprecated/` (отменено).

## Что внутри спринта

- Заголовок и дата создания.
- Цель — одним предложением.
- Объём работ — чеклистом.
- Критерий готовности — проверяемый (что должно работать).

## Договорённости проекта

- Дробим спринты по функционалу, не по слоям кода.
