# Исследование: порядок статус-айтемов и механика скрытия на Tahoe

Дата: 2026-06-05. Источник: практическая проверка ШАГА 1 вживую на macOS 26 Tahoe + perplexity + Apple docs + чтение исходника HiddenBar (dwarvesf/hidden). Это итог-факт по спринтам 01–02, основание для поправки к ADR 005.

## Краткий вывод

Спрятать чужие иконки в строке меню можно ТОЛЬКО выталкиванием шириной своего спейсера. Нативного API «скрыть чужую иконку» не существует. И второй важный факт: порядок своих `NSStatusItem` на Tahoe нельзя задать программно — он не зависит от порядка создания. Поэтому надёжность даёт не порядок, а проверка реальных координат перед раздуванием (guard).

## Что проверяли и что вышло

ШАГ 1: два своих айтема — видимый якорь `<` и спейсер-cutter. Спейсер раздувается до ~ширины экрана и выталкивает иконки левее себя за край. Якорь остаётся виден (он правее спейсера), клик по нему прячет/возвращает.

Сначала опирались на правило «спейсер создан вторым → встанет левее якоря». На практике это сломалось: спейсер вставал то левее, то правее якоря, и при раздувании уносил за край сам якорь («`<` улетел»).

## Доказанные факты

1. **Нет нативного API скрытия чужих иконок.** macOS не даёт приложению трогать чужие `NSStatusItem`. Единственный штатный способ убрать чужие иконки с глаз — раздуть свой спейсер так, чтобы он вытолкнул их за левый край экрана. Подтверждено через Apple docs, perplexity и чтение исходника HiddenBar.

2. **Порядок айтемов не детерминирован порядком создания и не управляется программно.** На macOS 11+ нет публичного API двигать свои или чужие айтемы. Порядок зависит от истории Cmd+drag пользователя и `setAutosaveName`, а не от того, кто вызвал `statusItemWithLength` первым.

3. **Решение — GUARD по реальным координатам.** Перед раздуванием спейсера сверяем X обоих айтемов через `button().window().frame().origin.x`. Раздуваем только если спейсер строго левее якоря. Иначе скрытие блокируем, якорь показывает `⚠<`, тултип просит поправить порядок Cmd+drag. Так в опасной конфигурации мы просто ничего не ломаем.

4. **Координаты доступны отложенно.** Сразу после старта `window()` айтема ещё нет — координаты появляются ~0.5–1с спустя, после стабилизации RunLoop. Поэтому стартовое авто-скрытие повешено на `NSTimer` 1с, а не сделано мгновенно.

5. **Ширину скрытия надо ограничивать.** Гигантские значения вроде `10000` дают патологию layout на Tahoe. Берём ширину экрана + запас 200, clamp 500..4000 — приём из HiddenBar.

6. **Позиции переживают перезапуск через `setAutosaveName`** (обоим айтемам). `defaults delete` на старте стирал эти позиции — убран.

## Отвергнутые подходы (чтобы не вернуться)

- **Один айтем, растущий «в себя».** Символ уезжает/пропадает при расширении — AppKit не держит контент справа в широком айтеме.
- **Пара айтемов с опорой только на порядок создания (без guard).** На Tahoe порядок ненадёжен: спейсер вставал правее якоря и выталкивал сам якорь.
- **Нативное скрытие чужих иконок через API.** Невозможно — такого API нет.

## Стек, подтверждённый рабочим

objc2 0.6.4, objc2-app-kit 0.3.2, objc2-foundation 0.3.2 (фичи `NSTimer`/`NSDate`), objc2-core-foundation. Делегат — через `define_class!`. Сборка — `bundle.sh` (.app + Info.plist `LSUIElement` + ad-hoc codesign), запуск — `ctl.sh`.
