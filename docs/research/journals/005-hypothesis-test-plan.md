# 005 — План тестов гипотез H1–H5 (вердикт по координатам)

**Дата:** 2026-06-07
**Метод:** systematic-debugging. Каждый тест меняет ОДНУ переменную, возвращает координаты.
**Вердикт по координатам (не по скриншоту):**
- `anchor.x ≈ 1700` (правый кластер) = ✅ якорь на месте
- `anchor.x = 0` или `< 100` (левый край) = ❌ сломано
- `spacer.x < anchor.x` и оба ≈ 1700 = ✅ правильный порядок

**Ограничение:** тесты последовательные (один menu bar на систему). Harness гоняет по очереди, парсит `/tmp/nook-debug.log` между запусками.

---

## Базовая линия (эталон, уже снято)

| Режим | spacer.x | anchor.x | вердикт |
|-------|----------|----------|---------|
| Прямой бинарь | 0 | 0 | ❌ |
| Из .app (есть bundle id) | 1700 | 1702 | ✅ |

---

## T1 — Восстановить регистрацию `.app` в LaunchServices, потом прямой бинарь
**Гипотеза:** H1. Если зарегистрировать `.app` (lsregister -f), прямой бинарь снова получит identity → координаты вернутся к 1700.
**Шаги:** `lsregister -f /Applications/Nook.app` → запустить прямой `./target/release/nook` → снять координаты.
**Ожидание H1-верно:** прямой бинарь даёт ~1700 (унаследовал identity).
**Ожидание H1-неверно:** всё равно 0/0 (identity не наследуется через реестр).
**Это ключевой тест — объясняет «раньше работало без .app».**

## T2 — Прямой бинарь с минимальным Info.plist рядом (без .app-обёртки)
**Гипотеза:** H1-вариант. Положить `Info.plist` в ту же папку что бинарь / выставить `__CFBundleIdentifier` env.
**Шаги:** `__CFBundleIdentifier=com.hypnosis.nook ./target/release/nook` → координаты.
**Ожидание:** если env даёт identity → 1700 (можно «без .app»!). Если нет → 0/0 (нужна реальная обёртка).
**Это тест на «можно ли всё-таки без .app».**

## T3 — Варьировать STARTUP_HIDE_DELAY (тайминг раздувания)
**Гипотеза:** H2. Когда координаты становятся валидными после старта?
**Шаги:** из .app, DIAG-таймер опрашивает координаты каждые 0.1с первые 3с. Снять, на какой секунде anchor.x становится ≈1700.
**Ожидание:** найти минимальную безопасную задержку. Если координаты валидны уже на 0.1с — H2 неверна (тайминг ни при чём). Если только после 0.5–1с — подтвердить задержку.

## T4 — Прямой бинарь БЕЗ guard (раздувание на x=0)
**Гипотеза:** H3. Что с раскладкой, если guard не блокирует при x=0.
**Шаги:** env `NOOK_NO_GUARD=1`, прямой бинарь, форсить toggle. Снять координаты якоря ДО/ПОСЛЕ.
**Ожидание H3-верно:** спейсер раздулся на 1928 от x=0, якорь утащило/перекрыло. Подтверждает «спейсер прятал всё».

## T5 — autosave: переживает ли перезапуск после Cmd+drag
**Гипотеза:** H4. autosave не сохраняется → проверить руками.
**Шаги:** из .app, снять начальные координаты → (вручную Cmd+drag нельзя автоматом, ОТЛОЖИТЬ на присутствие юзера) → перезапуск → defaults read.
**Статус:** требует ручного Cmd+drag — ПОМЕТИТЬ как «нужен юзер».

## T6 — Порядок создания: спейсер первым vs якорь первым
**Гипотеза:** H4. Порядок создания влияет на расстановку без autosave.
**Шаги:** env `NOOK_SPACER_FIRST=1`, из .app, снять координаты. Сравнить с дефолтом (якорь первым).
**Ожидание:** покажет, детерминирован ли порядок или случаен.

## T7 — SMAppService окружение (по логину) vs open
**Гипотеза:** H5. Запуск по автозапуску отличается.
**Статус:** требует регистрации SMAppService и релогина — ПОМЕТИТЬ как «нужен юзер / отложить».
