# UX-кейс от пользователя (сырое, 2026-06-05)

Зафиксировано из чата перед проектированием поведения. Вход для эксперт-проработки.

## Главный сценарий («бар снизу» = полу-IceBar)

1. Клик по `<` → снизу выезжает **бар-полоска** со своими ярлыками (примеры: Яндекс.Диск, Docker, Ollama, Time Machine). Клик `<` = «открыть бар».
2. Пользователь работает с приложениями в баре. **Клики по ярлыкам НЕ закрывают бар, пока на нём фокус.**
3. Закрытие бара:
   - повторный клик по `<`, ЛИБО
   - авто по таймеру: через ~5 секунд после потери фокуса (Bartender-style focus-out).

## Жалобы на текущий gate (что чинить после проектирования)

- Toggle технически работает (спейсер двигает иконки на Tahoe — фундамент жив).
- НО непонятно, что именно прячется: спейсер прячет только 1-2 соседние иконки, нет ясной границы «что лишнее».
- Нужно спроектировать поведение границы скрытия, не копируя nanobar вслепую.

## Открытые вопросы для эксперта

- Что считать «лишними» иконками: всё левее `<`? Две границы (всегда видны / скрываемые / всегда скрыты)? Как пользователь это понимает визуально?
- Связь «спейсер прячет иконки» и «бар-полоска со своими ярлыками»: это один жест (`<` и прячет, и открывает бар) или разное?
- Визуальный фидбек: как юзер понимает что спрятано и где граница.
- Механика «фокус на баре» без хаков (только свой NSPanel): как ловить focus-out для таймера 5с.

## Решения пользователя по вопросам (AskUserQuestion)

- Поведение скрытия: **«пусть эксперт предложит»** (не выбрал готовый вариант — нужна проработка).
- Роль полоски: `<` открывает бар снизу; клик/таймер 5с закрывает; фокус на баре держит открытым.
