# 006 — Режим редактирования с видимыми чертами + границы диалога настроек

**Статус: принято 2026-06-05.** Продолжение ADR-005, в линии «без хаков» (ADR-004). Спроектировано эксперт-агентом по идее пользователя «режим редактирования как у Bartender + черты».

## Context

Пользователь спросил, как выглядит спейсер и можно ли обозначить границу зоны скрытия явно. Предложил «Bartender-style»: режим редактирования с двумя видимыми чертами (`>` `<`), между которыми прячутся иконки, плюс возможно диалог «что видимо / что скрыть / что скрыть навсегда». Нужно было взять реализуемое и честно отсечь то, что упирается в хаки.

## Decision

### 1. Спейсер — это NSStatusItem, его вид меняется сменой title/length

Спейсер — обычный второй `NSStatusItem`. Сейчас в нём невидимый символ (zero-width space) → выглядит как пустой зазор. Сделать его видимым — это просто сменить `title`/`length`/`alphaValue`, без новых объектов и без хаков.

### 2. Режим редактирования (STATE_4) — берём

- **Обычный режим:** левая граница зоны B невидима (спейсер пустой, width≈6), виден только `‹`.
- **Режим редактирования:** спейсер получает видимый заголовок `┊` (alpha≈0.4), `‹` меняется на `›`/`›─` — обе границы зоны B видны. Пользователь Cmd+drag раскладывает иконки между чертами.
- **Вход/выход:** правый клик по `‹`/`›` → контекстное меню → «Редактировать зону скрытия» / «Готово». Esc тоже выходит. (Hotkey — опционально, потом.)
- **Подсказка:** tooltip «Перетащи иконки Cmd+drag между › и ┊». Онбординг первого запуска: один раз показать STATE_4 с подсказкой.
- Во время редактирования бар НЕ открывается (левый клик игнорируется). Из STATE_2/3 в редактирование нельзя — сначала закрыть бар.

### 3. Контекстное меню (правый клик по разделителю)

```
✎ Редактировать зону скрытия   (или «Готово» в режиме редактирования)
──────────────
Ярлыки бара…
Настройки…
──────────────
Выйти
```

### 4. Диалог настроек — предмет управления НЕ чужие иконки, а поведение clearbar

Секции:
- **Зоны скрытия** — пояснение + кнопка «Войти в режим редактирования». (Опц. потом: третья зона «всегда скрытые» через третий спейсер.)
- **Бар-полоска** — список СВОИХ ярлыков (наши данные): добавить (NSOpenPanel → .app), удалить, переупорядочить; размер бара.
- **Поведение** — таймер автоскрытия (выкл/3/5/10с), hotkey открытия бара, запуск при логине.
- **О программе** — версия, ссылка.

Всё на своих данных, без хаков.

## Жёсткая граница (что НЕВОЗМОЖНО — закон macOS)

**Список реальных чужих иконок с тумблером hide на каждую — нереализуем без хаков.** Мы не знаем ни имя, ни картинку, ни количество чужих `NSStatusItem` — это требует Screen Recording + CGWindowList. Именно так делает Bartender, и именно это ломается на Tahoe. Поэтому:

- НЕ обещаем «вот список твоих иконок, включи/выключи каждую».
- НЕ воспроизводим диалог Bartender со списком иконок.
- Воспроизводим UX-ощущение Bartender (зоны, редактирование, бар) другими средствами — нашими разделителями и своими ярлыками.

«Переписать Bartender бронебойно» буквально невозможно: Bartender делает что делает именно через хаки. Наш путь даёт похожий опыт без них.

## Полный автомат состояний (с STATE_4)

```
STATE_1: Shown × Closed × Normal       — базовое
STATE_2: Hidden × Open × Normal        — бар открыт, спейсер раздут
STATE_3: Hidden × ClosingPending       — бар открыт, таймер 5с
STATE_4: Shown × Closed × Editing      — режим редактирования, обе черты видны

STATE_1 + клик_‹/hotkey                  → раздуть спейсер + показать NSPanel → STATE_2
STATE_2 + клик_›/hotkey                  → скрыть NSPanel + сжать спейсер     → STATE_1
STATE_2 + mouse_exit/resignKey           → таймер 5с                          → STATE_3
STATE_2 + клик_ярлыка                    → NSWorkspace.open(app)              → STATE_2
STATE_3 + mouse_enter                    → отменить таймер                    → STATE_2
STATE_3 + клик_›/hotkey                  → отменить таймер + закрыть          → STATE_1
STATE_3 + таймер_истёк                   → закрыть                           → STATE_1
STATE_1 + правый_клик «Редактировать»    → видимые черты + подсказка          → STATE_4
STATE_4 + правый_клик «Готово»/Esc       → скрыть черту                       → STATE_1
STATE_4 + левый_клик_‹                   → игнорировать (не открывать бар)
STATE_2/3 + «Редактировать»              → запрет, тост «сначала закрой бар»
```

## Alternatives

- **Диалог со списком чужих иконок + тумблеры (полный Bartender).** Отвергнут: требует Screen Recording/CGWindowList — хаки, ломается на Tahoe (ADR-004).
- **Только невидимый спейсер без режима редактирования.** Отвергнут: пользователь не видит границу зоны скрытия (исходная жалоба).

## Consequences

- Автомат расширяется на STATE_4 → влияет на `app.rs` (оркестрация) и обработку правого клика (контекстное меню).
- Спейсер должен уметь два вида: невидимый (Normal) и видимая черта `┊` (Editing).
- Диалог настроек (T5d) управляет своими данными, не чужими иконками — упрощает спринт.
- MVP: режим редактирования с чертами + контекстное меню + диалог настроек (ярлыки/таймер/hotkey/автозапуск). Потом: третья зона, анимации, мультимонитор, hotkey для STATE_4.
