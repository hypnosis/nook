# 001 — Стек: Rust + objc2

## Context

clearbar управляет строкой меню macOS. Строкой меню в macOS можно управлять только через нативные API Apple (`NSStatusItem`, `NSStatusBar` из AppKit) — другого пути нет. Нужно выбрать язык и способ выхода на эти API. Требования: предсказуемая работа без подвисаний, минимум лишних зависимостей, лёгкое приложение.

## Decision

Пишем на **Rust** и ходим в AppKit через **objc2** — тонкую FFI-обёртку над Objective-C runtime. objc2 даёт прямой доступ к `NSStatusItem`/`NSStatusBar`, не таща за собой ничего лишнего. Rust выбран за предсказуемость памяти (нет пауз сборщика мусора, строгая модель владения) и лёгкость итогового бинарника.

## Alternatives

- **Swift + AppKit.** Самый родной путь для macOS. Отвергнут: пользователь хочет именно Rust.
- **Tauri.** Отвергнут: тащит WebView — лишняя тяжёлая зависимость для утилиты, которой нужно лишь нарисовать узкую полоску с иконками.
- **Чистый Rust без зависимостей.** Невозможен: строка меню доступна только через нативные Apple API, без FFI-моста к AppKit обойтись нельзя.

## Consequences

- Часть кода — небезопасный FFI к Objective-C; границу с `unsafe` держим тонкой и изолированной.
- Привязка к objc2 и к версии AppKit в macOS 26 Tahoe.
- Бинарник лёгкий, запуск быстрый, без рантайма WebView и без GC.
