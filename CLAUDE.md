# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

`clearbar` — лёгкая нативная утилита для macOS, которая прячет лишние иконки в строке меню (menu bar). Аналог HiddenBar и Ice, написанный для себя. Лишние иконки выталкивает за край спейсер, а при клике на разделитель `<` снизу выезжает полоска-бар со своими ярлыками.

> **Путь выбран — полу-IceBar (принято 2026-06-05, см. ADR 004).** Изначально хотели «второй ряд» со скрытыми иконками И «никаких хаков». Исследование показало: зеркало чужих иконок без хаков невозможно (требует Screen Recording + CGEvent, ломается на Tahoe). Решение: второй ряд = отдельное окно-полоска (`NSPanel`) со СВОИМИ плашками-ярлыками, а не снимками чужих иконок. Скрытие делает спейсер. Без Screen Recording, без `CGEvent`, без Accessibility. Цена — полоска показывает свои ярлыки, а не зеркало системных иконок.

## Зафиксированные технические решения

- **Стек: Rust + objc2.** objc2 — это тонкая обёртка над нативным AppKit (`NSStatusItem`, `NSStatusBar`), потому что строкой меню в macOS можно управлять только через родные API Apple. «Чистый Rust без зависимостей» здесь невозможен. Rust выбран за предсказуемость памяти и лёгкость. Swift и Tauri отвергнуты (см. ADR 001).
- **Таргет: только macOS 26 Tahoe.** Старые версии macOS не поддерживаем.
- **Нет хакам Bartender (приоритет — надёжность на Tahoe).** Мы НЕ перехватываем чужие иконки через скриншот строки меню (Screen Recording) и Accessibility/CGEvent — именно из-за этого Bartender глючит и ломается на каждом обновлении macOS. Закон macOS: приложение управляет только своими `NSStatusItem`, чужие иконки штатно трогать нельзя. Надёжный путь — спейсер: свой невидимый разделитель раздувается и выталкивает иконки за край экрана (как HiddenBar). См. ADR 003 (с поправкой) и ADR 004.
- **UX «второй ряд» — полу-IceBar.** Второй ряд = отдельное окно-полоска (`NSPanel`) со своими плашками-ярлыками под строкой меню, а НЕ зеркало чужих системных иконок. Полный Ice (реальные снимки чужих иконок) отвергнут — хрупок на Tahoe и требует хаков. См. ADR 002 и ADR 004.
- **MVP:** прятать/показывать иконки (спейсер) ✅ → автоскрытие ✅ → контекстное меню ✅ → полоска-бар своих ярлыков (полу-IceBar) `want fix` → настройки с hotkey и автозапуском при логине `want fix` → современная иконка и стиль `want fix`. Реализовано в v0.1.0: скрытие, автоскрытие, меню. Остальное (`want fix`) — бэклог.

## Команды

Кода пока нет — это стартовый каркас (бутстрап). Реальные команды появятся после `cargo init` в рамках первого спринта:

- `cargo build` — сборка
- `cargo run` — запуск
- `cargo test` — тесты
- `cargo build --release` — релизная сборка

Таргет сборки — только macOS 26 Tahoe.
